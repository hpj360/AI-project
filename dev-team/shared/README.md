# DevTeam 共享文件区

这是团队协作的共享文件区，所有需要跨角色共享的文档都应存放在这里。

## 目录结构

```
shared/
├── requirements/     # 需求文档 (BA负责)
├── architecture/     # 架构设计 (Architect负责)
├── designs/          # 设计稿 (UI负责)
├── api-specs/        # API规范 (Backend负责)
├── test-reports/     # 测试报告 (QA负责)
├── decisions/        # 技术决策记录 (ADR)
└── docs/             # 其他共享文档
```

## 使用规范

### 文件命名
- 需求文档：`REQ-<编号>-<名称>.md`
- 架构文档：`ARCH-<编号>-<名称>.md`
- 设计文档：`DESIGN-<编号>-<名称>.md`
- API规范：`API-<模块>-<版本>.yaml`
- 测试文档：`TEST-<编号>-<名称>.md`
- 决策记录：`ADR-<编号>-<标题>.md`

### 访问权限
- 所有角色都可以读取共享文件
- 各目录由对应角色负责维护
- 重大变更需要通知相关角色

### 版本控制
- 重要文档变更需记录更新日志
- 使用Git进行版本管理
- 重大变更创建新版本

## 注意事项
1. 不要在共享区存放敏感信息
2. 文件名使用英文，内容可使用中文
3. 大文件考虑使用外部存储并在此放置链接
